Fonctionnalités Principales

Découvrez comment ClamAVClient allie la puissance industrielle du moteur de détection ClamAV à une ergonomie moderne.

Protection Temps Réel

Surveillance active des répertoires sensibles. Tout fichier écrit ou modifié dans vos répertoires configurés est immédiatement analysé par le moteur.

Mises à Jour Automatiques

Mise à jour périodique des bases de signatures de virus Cisco Talos en tâche de fond. Votre protection est toujours armée contre les dernières menaces.

Quarantaine Sécurisée

Les fichiers infectés sont immédiatement déplacés et isolés dans une zone de quarantaine sécurisée cryptographique, évitant toute exécution accidentelle.

Intégration clamd optionnelle

Basculez à tout moment vers un démon `clamd` local ou réseau pour diviser la durée de scan par 10 en conservant le moteur chargé en mémoire RAM.

Multi-Plateforme Natif

Compilé de manière native pour Windows (MSI), macOS (DMG Intel/M1/M2) et Linux (DEB, RPM et AppImage) grâce au framework ultra-léger Tauri.

Notifications Système

Alertes visuelles immédiates sur votre système d'exploitation dès qu'une menace est identifiée ou qu'une quarantaine automatique est effectuée.

Documentation Technique

Apprenez à installer, configurer et participer au développement du client léger de ClamAV.

Installation pas à pas

Windows

Téléchargez et exécutez le fichier d'installation .msi. Le programme installe le client et crée un raccourci de démarrage automatique configuré avec élévation UAC.

# Si vous préférez installer via winget (prochainement disponible)
winget install ClamAVClient

macOS

Montez le fichier .dmg téléchargé, puis glissez l'icône de ClamAVClient dans votre dossier Applications. Au premier démarrage, l'application configurera un service launchd pour la mise à jour des signatures.

Linux (Debian / Ubuntu / Linux Mint)

Installez le fichier .deb avec dpkg puis corrigez les éventuelles dépendances manquantes avec apt.

sudo dpkg -i clamavclient.deb
sudo apt-get install -f

Linux (Fedora / RHEL / Rocky Linux)

sudo rpm -i clamavclient.rpm

Linux AppImage (Toutes distributions)

chmod +x ClamAvClient.AppImage
./ClamAvClient.AppImage

Configuration des Privilèges

Pour être capable d'analyser l'intégralité du système de fichiers et de surveiller en temps réel l'ensemble des écritures, l'antivirus requiert des privilèges administratifs élevés :

  • Sous Windows : Le manifeste d'application généré par Tauri demande automatiquement le consentement UAC de l'administrateur à chaque ouverture de l'application.
  • Sous Linux / macOS : L'application vérifie son ID d'utilisateur au démarrage. Si l'application n'est pas lancée en tant que root, elle tente d'utiliser les agents d'authentification graphique de votre bureau (comme pkexec pour Polkit sur Linux, ou osascript sous macOS) pour réclamer l'élévation.

Si aucun système d'élévation graphique n'est disponible ou si le consentement est rejeté, le client se fermera par mesure de sécurité.

Compiler le projet

ClamAVClient est écrit en Rust (backend Tauri) et en Javascript Vanilla (frontend). Pour le compiler localement, vous devez installer Node.js, Rust (via rustup), et le moteur ClamAV local.

1. Cloner le dépôt de code

git clone https://github.com/AnARCHIS12/ClamAVClient.git
cd ClamAVClient

2. Installer les dépendances NPM

npm install

3. Préparer le runtime ClamAV embarqué

Cette étape télécharge ou copie les fichiers binaires requis par ClamAV sur votre système d'exploitation actuel pour les incorporer au bundle de l'application desktop.

npm run prepare:clamav

4. Lancer le serveur de développement élevé

# Lance l'application avec les privilèges sudo/administrateur requis
npm run dev:elevated

5. Compiler l'application finale pour votre OS

npm run build:desktop

Fonctionnalités et variables avancées

Option Daemon (clamd)

Par défaut, l'application charge le moteur de scan en mémoire à chaque analyse, ce qui est économe en RAM mais peut être lent à démarrer. En activant l'option clamd dans les configurations, l'application se connecte au démon résident système (souvent configuré sur le port 3310).

Structure des Logs d'analyse

Chaque scan consigne son rapport au format standardisé JSON dans le dossier de données de l'application :

  • Windows : %APPDATA%/ClamAVClient/logs/
  • Linux : ~/.config/clamavclient/logs/
  • macOS : ~/Library/Application Support/ClamAVClient/logs/

Questions Fréquentes (FAQ)

Tout ce que vous devez savoir sur ClamAVClient et le moteur de scan sous-jacent.

Oui, ClamAVClient embarque une base de signatures initiale au moment de l'installation pour assurer une analyse hors-ligne. Cependant, pour récupérer les nouvelles signatures (mises à jour quotidiennes de la communauté et de Cisco Talos), une connexion Internet est nécessaire pour le service Freshclam.
Lorsqu'une menace est détectée, le fichier suspect est déplacé dans un dossier sécurisé géré par ClamAVClient. Il est privé de ses droits d'exécution d'origine pour empêcher tout lancement accidentel. Vous pouvez à tout moment restaurer le fichier à son emplacement d'origine ou le détruire définitivement via l'onglet Quarantaine.
Les antivirus ont besoin de parcourir des zones réservées du système d'exploitation (fichiers système, bibliothèques système partagées) pour débusquer les programmes malveillants. Sans droits élevés, l'antivirus serait incapable d'analyser ces dossiers critiques et de surveiller en temps réel l'écriture des fichiers par d'autres processus système.
En mode autonome, l'application charge en mémoire vive les 150 Mo de base de signatures de virus à chaque scan, ce qui libère la RAM après l'analyse mais ralentit le démarrage de celle-ci. En mode `clamd` (daemon), un processus d'arrière-plan conserve en permanence la base de signatures en mémoire vive. Les scans sont alors quasi instantanés, mais consomment environ 150 à 200 Mo de RAM résidente.
